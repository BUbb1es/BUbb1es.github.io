常见Web安全问题

发表于2022-07-08|更新于2022-07-21
|字数总计:3.3k|阅读时长:12分钟|阅读量:
  • 相关信息搜索
  1. 快速查询相关篡改网页关键字
    Hacked by
  2. 搜索引擎语法
    Intitle：关键字 标题含
    Intext：关键字 正文内含
    Site：域名 某域名或某子域名下所有网页
  3. URL查看源码
    view-source:URL

    • 手段
      通过微信、QQ、邮件、二维码等发送钓鱼链接
    • 方法
      诱惑标题、仿冒网站、色图二维码内嵌
    • 目的
      用户输入相关的私密信息(密码、个人隐私等)从而被钓鱼后台捕获
    • 流程
    1. 攻击网页(技术手段多)
    2. 修改网页(挂马(上传后门)、暗链)

    • 页面篡改
      表现就是现页面跟原页面有差别，被入侵修改成其他内容(通常是秀技或威胁)

    • 植入暗链 (大部分)
      原有页面内植入黑灰产（色情、博彩、医疗、游戏）的链接(隐藏网页内)

    • 主要受众
      日常上网用户(X)
      针对搜索引擎的爬虫机器人(√)

    • 目的
      提高链接的搜索引擎排名(SEO)

    • Webshell(挂马,上传植入 )
      后门工具(多以动态脚本出现)，基本是root权限
      形式：asp/php/jsp 文件+网页混合 浏览器运行
      控制流量一般跟正常流量混合(隐蔽难发现)
      大马→功能齐全
      小马→一句话木马(特定功能，较简单)

    跨站脚本攻击(Cross Site Scripting)，前端(HTML)注入->插入恶意脚本盗取cookie等

    • 目的
      盗取个人隐私、钓鱼、蠕虫等

    • 分类
      ①存储型(被动)
      访问含有XSS的网页时触发
      XSS脚本会通过浏览器访问提交到web应用程序时恶意写入SQL数据，并返回恶意SQL数据到浏览器上(保存到数据库内 )

    XSS数据存储：数据库

    输出XSS数据：后端Web应用程序(HTTP响应内)

    简易流程： ①访问应用→植入XSS脚本(数据库 )
    ②用户访问含XSS页面→触发XSS脚本
    例：
    论坛留言→植入XSS代码到数据库→浏览器触发XSS



    ②反射型(主动)
    主动访问携带XSS脚本的链接时触发
    可以通过恶意链接看到明显的XSS脚本特征
    用户访问携带XSS脚本链接，浏览器向web应用程序发送请求，然后web应用程序将url的XSS脚本数据写入到响应页面并返回浏览器，渲染响应页面并触发XSS

    XSS数据存储：URL

    输出XSS数据：后端Web应用程序(HTTP响应内)

    简易流程：访问含XSS脚本的链接→URL 提取XSS脚本所需数据并触发
    通过后端Web应用程序将XSS脚本写入响应页面，当浏览器渲染响应页面时触发XSS
    例：
    URL后添加XSS代码



    ③DOM型(主动)
    类似反射型

    XSS数据存储：URL

    输出XSS数据：前端JavaScript(动态构成的DOM节点)

    区别：访问含XSS脚本的链接→URL提取XSS脚本所需数据→写入DOM 触发XSS
    通过前端Java Script将XSS脚本写入DOM触发XSS
    例：
    通过登录界面提交错误信息→若URL返回hash并测试添加内容可行时，跟其后植入XSS代码

    查看源码时不能查看XSS代码，可通过查看hash的响应代码(间接)

    跨站请求伪造(Cross Site Request Forgery)，利用用户已登录red 的身份完成非法操作(不知情)

    • 前提：用户已登录并点击相关的CSRF攻击链接(诱人标题：XXX劲爆新闻)

    • 目的：
      以用户的名义发送钓鱼邮件、垃圾消息，盗取用户账号，添加系统管理员，甚至于购买商品、虚拟货币转账等

    • 简易流程
      黑客建成恶意网页→用户主动访问该页面→进行恶意操作(转账)
      利用cookie(浏览器发送任何请求会自动添加已有cookie)→后端识别到cookie，进行操作(转账)

    网页内嵌攻击代码：(登陆后，点击网页内带有攻击代码的图片或链接即可自动完成转账)


    若想更隐蔽，可以通过iframe(外层网页)然后基于iframe内的内容进行攻击代码的编辑(内层网页)，没有明显的form提交跳转
    iframe特点：可创建包含另一个文档的内联框架

    • 特点
      嵌套
      内部不可见
      隐蔽性高
      骗取用户点击操作(UI-覆盖攻击)

    • 总体原理
      主要利用iframe可包含文档的内联框架的特点(也可利用其他标签的属性特点)
      覆盖不可见的iframe或其他标签的属性误导用户去点击进行一系列的危险操作(转账)的攻击行为

    • 流程
      先插入具有诱惑力的图片(色图)或游戏
      再插入iframe标签内嵌隐藏网页(需调节当前网页与内嵌iframe标签网页的属性)

    将通常可信的URL与恶意URL混淆在一起
    用户点击伪装成可信任的恶意网站进行一些私密操作，从而被黑客攻击的行为

    • 实现方式
      1.Header头跳转

      2.Javascript跳转

      3.META跳转

    • 流程
      1.构建恶意链接→发布给用户(论坛、聊天软件等)
      2.访问Web应用→服务器/浏览器解析
      3.跳转恶意网站(有迷惑性，与正常网站相似)

    https://(可信链接，如:www.baidu.com)?url=(恶意链接：www.evil.com)

    通过官网跳转到伪装成官网的网站更有迷惑性(或转为短链接)

    http://mail.qq.com/cgi-bin/mail_spam?action=check_link&spam=0&url=http://t.cn/RVTatrd

    万能密码(没有密码时)：在用户名的文本框中输入字符串：“admin’ — ”
    利用SQL注入漏洞可访问修改数据或利用潜在的数据库漏洞进行攻击

    • 本质
      数据和代码未分离(可控制的数据被当作代码执行)

    • 必备条件
      1.可控制输入的数据
      2.服务器要执行的代码拼接了控制的数据(基于1)

    • 操作流程
      用户登录时→POST请求→服务器解析(PHP代码)→获取请求参数(用户名和密码，密码会经过一次MD5处理)

    正常登录

    万能密码登录(SQL注入的利用方式之一)

    • 注入执行步骤(类似正常请求数据)
      (构造SQL查询后)
      1.获取用户请求参数
      2.拼接到代码中(关键)
      3.SQL语句按照黑客构造参数的语义执行成功

    • 利用
      获取系统版本: “username=admin’ and 1=2 union select version() — ”
      获取当前数据库用户：“username=admin’ and 1=2 union select user() — ”
      也可查询当前数据库名称(DB)和数据表(Table)以及详细内容

    • 危害
      获取数据库信息
      1.管理员后台用户名和密码
      2.其他数据库敏感信息(用户名、密码、手机号码、身份证…)
      3.整个数据库(脱裤)
      4.获取服务器权限
      5.植入Webshell，获取服务器后门
      6.万能密码
      …

    • 常见文件操作和对应相关漏洞
      用户
      文件上传(图像、附件等)
      漏洞利用：
      ->上传Webshell
      ->上传木马
      ->内网渗透
      …
      漏洞条件：可上传可执行脚本&脚本要拥有执行权限(文件处理不当)

    文件下载(图像、附件等)
    漏洞利用：
    ->下载系统任意文件
    ->下载程序代码
    ->下载数据库文件
    …
    漏洞条件：未验证下载文件格式&未限制请求的路径(文件处理不当)

    • 文件上传漏洞流程
      构建恶意文件(php文件,内含漏洞证明代码)->上传文件->网页内查看图片地址->返回运行结果

    • 文件下载漏洞流程
      获取下载请求地址->修改地址(关键)尝试不同下载链接->下载想要的内容(代码&敏感内容)

    • 文件包含漏洞
      重复代码写入一个文件(方便引用，美观) ->(txt/php/jpg)恶意文件注入
      Attention：
      allow_url_fopen = On(是否允许打开远程文件)
      allow_url_include = On(是否允许包含远程文件)

    类型：
    本地文件包含->本地服务器的文件
    远程文件包含->通过发送http请求或其他请求来运行的远程文件

    常见文件包含函数：
    include()
    require()
    include_once()
    require_once()
    …
